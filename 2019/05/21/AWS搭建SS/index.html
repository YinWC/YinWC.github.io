薅羊毛

网址：http://ctuair.com/ (现在似乎凉了)
最新网址：https://2vcc.com/ 涨价了2333
之前可以成功购买，现在一直在用：

购买后，使用AWS卡号注册国际版的账户（网址：https://aws.amazon.com/）
用户名用英文像真名一样才行
最后将购买的卡作为默认卡号，如果不行还可以去联系管理员（官网上有QQ号）

搭建SS

登录进去点击首页菜单->服务->EC2,注意选择“东京”的节点。

然后选择一个系统映像，实例类型用推荐的，点击审核并且启动。


之后会弹出来一个创建密匙对的框，在第一个下拉框中，选择“创建新密匙对”。在第二个框中输入你的密匙对的名字，这里取名叫“ywc1”。点“下载密匙对”，会下载一个叫“ywc1.pem”的文件，把文件保存在一个绝对安全的地方。

此时我们已经创建了一个实例了.下面我们启动实例, 服务器就已经创建并启动了.

点击连接，会出现：

可以看到服务器地址：ubuntu@ec2-13-230-225-115.ap-northeast-1.compute.amazonaws.com
账户：ubuntu
使用MobaXterm连接服务器

直接连接，会发现有错误

导入密钥即可解决：Use private key->选择保存的pem文件

即可连接服务器
使用SS脚本：ss脚本 wget --no-check-certificate -O shadowsocks-libev.sh https://raw.githubusercontent.com/uxh/shadowsocks_bash/master/shadowsocks-libev.sh && bash shadowsocks-libev.sh
会显示[Error] This script need to run as root!
sudo bash shadowsocks-libev.sh即可
下面就是选择端口、设置密码等配置SS的常规操作了，一般除了密码默认即可。

配置安全策略组


点击实例->安全组->launch-wizard-3 编辑入站

新增加SS端口2199和所有流量入站（虽然不太安全，这里我主要为了ping的通2333）

参考文章：
https://docs.aws.amazon.com/zh_cn/AWSEC2/latest/UserGuide/get-set-up-for-amazon-ec2.html#create-a-base-security-group
https://www.cnblogs.com/luosiding/p/8480184.html
